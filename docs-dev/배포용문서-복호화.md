# HWP 배포용(distribution) 문서 복호화 로직

한글(HWP) "배포용 문서"는 본문을 **암호화**해 저장하고 읽기 전용으로 잠근다.
한글 프로그램은 내부에 복호화 절차를 갖고 있어 화면에 정상 표시하지만,
외부 도구(파이썬 등)는 본문 바이트가 암호문이라 그냥은 읽지 못한다.
이 문서는 그 본문을 프로그램으로 추출하기 위해 구현한 복호화 절차를 정리한다.

> 구현체: [`scripts/hwp_decrypt.py`](../scripts/hwp_decrypt.py)

---

## 1. 파일 구조 (OLE 복합 문서)

HWP 5.0 파일은 OLE2 복합 문서다. 주요 스트림:

| 스트림 | 역할 | 배포용 문서에서 |
|--------|------|-----------------|
| `FileHeader` | 시그니처(`HWP Document File`) + 플래그 | 압축/암호화/**배포용** 플래그 |
| `DocInfo` | 문서 정보(글꼴·스타일 등) | |
| `BodyText/SectionN` | 일반 문서의 본문 | **거의 비어 있음**(껍데기) |
| `ViewText/SectionN` | 배포용 본문 | **AES로 암호화된 실제 본문** |
| `PrvText` | 미리보기 텍스트(UTF-16LE) | 첫 페이지 일부만 평문 |

### FileHeader 플래그 (오프셋 36, uint32 LE)

```
bit 0 : 압축(compressed)
bit 1 : 암호화(password)
bit 2 : 배포용(distribute)   ← 이 비트가 켜지면 본문이 ViewText 에 암호화되어 들어감
```

배포용 문서는 `BodyText` 가 사실상 비어 있고, 실제 본문은 `ViewText/SectionN` 에 있다.
그래서 일반 HWP 추출기로 열면 빈 결과가 나온다.

## 2. ViewText 스트림 레이아웃

```
┌────────────┬───────────────────────┬───────────────────────────────┐
│ 레코드 헤더 │  256바이트 payload    │  AES-128-ECB 로 암호화된 본문   │
│  (4바이트)  │ (DISTRIBUTE_DOC_DATA) │  (16바이트 배수)               │
└────────────┴───────────────────────┴───────────────────────────────┘
```

### 레코드 헤더 (uint32 LE) 비트 필드

```
tagID = header & 0x3FF            (10비트)  → 28 = HWPTAG_DISTRIBUTE_DOC_DATA
level = (header >> 10) & 0x3FF    (10비트)
size  = (header >> 20) & 0xFFF    (12비트)  → 256
```

`size == 0xFFF` 이면 다음 4바이트가 실제 크기(확장 크기)다. (본문 레코드 순회에서 처리)

---

## 3. 256바이트 payload 복호화 (LCG + XOR)

payload 앞 4바이트가 **시드(seed, uint32 LE)** 이고, 이 시드로 만든
**MSVC CRT `rand()` 호환 선형합동난수(LCG)** 키스트림으로 XOR 복호화한다.

### LCG (MSVC `rand()` 호환)

```
seed = (seed * 214013 + 2531011) mod 2^32
출력  = (seed >> 16) & 0x7FFF
```

### XOR 복호화 루프 (런렝스 방식)

같은 키 바이트를 1~16바이트씩 연속 적용한다. **앞 4바이트(시드)는 건드리지 않는다.**

```python
seed = u32_le(payload[0:4])
lcg  = LCG(seed)
n = 0
for i in range(256):
    if n == 0:
        key = lcg.rand() & 0xFF          # 새 키 바이트
        n   = (lcg.rand() & 0x0F) + 1    # 이 키를 적용할 길이 (1~16)
    if i >= 4:                            # 시드 4바이트는 보존
        payload[i] ^= key
    n -= 1
```

## 4. AES-128 키 추출

복호화된 256바이트 payload에서 키 위치를 구한다.

```
offset = 4 + (payload[0] & 0x0F)     # payload[0] = 시드의 최하위 바이트
aes_key = payload[offset : offset + 16]   # 16바이트 = AES-128
```

---

## 5. 본문 복호화 + 압축 해제

```
body      = ViewText[4 + 256 : ]          # 헤더+payload 이후 전체
decrypted = AES-128-ECB(aes_key).decrypt(body)   # ECB, 패딩 없음 (16배수 보장)
section   = raw_inflate(decrypted)        # FileHeader 압축 비트가 켜져 있으면 (-15, zlib raw deflate)
```

`section` 은 일반 HWP 섹션과 동일한 **레코드 스트림**이다.

## 6. 본문 레코드에서 텍스트 추출

섹션은 `[헤더(4B)][payload(size)]` 레코드의 연속이다. 그중
`HWPTAG_PARA_TEXT`(= `HWPTAG_BEGIN(0x10) + 51` = **67**) 레코드가 문단 텍스트다.

### PARA_TEXT payload 디코딩 (UTF-16LE + 인라인 제어문자)

2바이트(wchar) 단위로 읽으며 제어문자를 처리한다.

| 코드(wchar) | 의미 | 처리 |
|-------------|------|------|
| 10, 13 | 줄바꿈 | `\n` |
| 0, 24~31 등 단일 제어 | 무시 | 2바이트 건너뜀 |
| 1,2,3,11,12,14~18,21~23 (확장) / 4~9,19,20 (인라인) | 표·각주 등 | **16바이트(8 wchar) 건너뜀** |
| 0xD800~0xDBFF | UTF-16 상위 대리(보충면 한자) | 다음 wchar와 합쳐 4바이트로 1글자 |
| 그 외 | 일반 문자 | `chr(code)` |

---

## 7. 전체 흐름 요약

```
HWP(OLE) ─▶ ViewText/SectionN
            ├─ 헤더(4B): tag=28, size=256 확인
            ├─ payload(256B): LCG(seed)+XOR 복호화
            │                 └─ aes_key = payload[4+(payload[0]&0xF) : +16]
            ├─ body: AES-128-ECB 복호화
            └─ raw-inflate ─▶ 섹션 레코드 ─▶ PARA_TEXT(67) ─▶ 본문 텍스트
```
